## 实现细节

### 1. `perpredict.py` 改动

#### 1.1 strategy 分发 (`_PD_METHOD_REGISTRY`)

```python
# Module-level (after CORE_PD_* 常量区)
_PD_METHOD_REGISTRY = {
    'core_alchabitius': 'getPrimaryDirectionByZCoreKernel',
    'horosa_legacy': 'getPrimaryDirectionByZLegacy',
    'placidus': 'getPrimaryDirectionByZPlacidus',
}

def getPrimaryDirectionByZ(self):
    method = getattr(self.perchart, 'pdMethod', 'core_alchabitius') or 'core_alchabitius'
    handler_name = _PD_METHOD_REGISTRY.get(method) or _PD_METHOD_REGISTRY['core_alchabitius']
    handler = getattr(self, handler_name)
    pdlist = handler()
    self.appendDateStr(pdlist)
    return pdlist
```

值是方法名字符串(string-based 延迟绑定),`getattr(self, name)` 在调用时再 resolve。未知 method fallback 到 Alcabitius — 这是铁律①的关键守卫。

#### 1.2 timeKey 统一抽象 (`_pdTimeKeyScale` + `STATIC_TIME_KEY_SCALES`)

```python
    ...
```

替换 `perpredict.py:730-731` 原 Naibod 硬编码:

```python

```

#### 1.3 Placidus kernel (`getPrimaryDirectionByZPlacidus`)

复用 flatlib `PrimaryDirections(chart).getList()` 默认 Placidian semi-arc 算法,提取 `'Z'` 标记 row。**不带 Alcabitius 的 ML 修正层**(那层是 v2.5.3 byte-perfect 默认路径专属)。

```python
def getPrimaryDirectionByZPlacidus(self):
    chart = self.perchart.getChart()
    pd = PrimaryDirections(chart)
    pdlist = []
    for item in pd.getList(self.perchart.pdaspects):
        if len(item) > 3 and item[3] == 'Z':
            pdlist.append(item)
    return pdlist
```

### 6. 校准脚本 `scripts/fit_pd_constants.py`

自研拟合脚本,基于内部校准语料(`pd_calibration_corpus/option_probe/`)反推 static time-key 常量。算法:对每个 time-key 输出流,从任意两 row 联立 `scale = (arc_2 - arc_1) * 365.24219879 / (jd_2 - jd_1)`,取所有 row 对的中位数。归一化到 `Ptolemy == 1.0` 基线(外部参考的 Ptolemy 拟合值带微小 epoch offset,需归一化才能得到标准比例)。

输出建议常量到 stdout(操作员手动复制进 `STATIC_TIME_KEY_SCALES`,脚本不自动写文件)。
