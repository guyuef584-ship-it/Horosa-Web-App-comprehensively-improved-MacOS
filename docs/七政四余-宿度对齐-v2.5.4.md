# 七政四余 二十八宿度·自有恒星案三制对齐 — 技术文档 (v2.5.4 本地)

## 背景 / 问题

七政四余盘的「宿度制」下拉(`doubingSu28`)有五种:

| 值 | 名称 | 常量 (perchart.py) |
|---|---|---|
| 0 | 荀爽19年测量 | SU28_MODE_REAL |
| 1 | 斗柄定房法 | SU28_MODE_DOUBING |
| 2 | 回归今制 | SU28_MODE_MOIRA_CURRENT |
| 3 | 回归开禧（开禧历）| SU28_MODE_MOIRA_KAIXI |
| 4 | 恒星制（郑式）| SU28_MODE_ZHENG_SIDEREAL |

> 标签更名(2026-06-02):mode 3 原 UI 标签「回归古制（开禧）」改为「**回归开禧（开禧历）**」。原因见文末「古制≠开禧」澄清。

**用户报的 bug**:「回归今制」宿度算错,与参考软件差很多(七政整体偏 ~18°,如日落在「轸」应为「翼」)。

**根因**:`getMoiraFixedStarSu28` 对「回归今制」**直接用了冻结的 `MOIRA_CURRENT_STELLAR_DEGREES`(15.9, 26.3 …)、零岁差**。但这组 15.9… 实为**郑氏恒星基值(sidereal)**,不是今制。今制应是 **28 距星的活体 tropical 黄经**(随盘日期岁差,逐宿不均匀)。两者差一个 ayanamsha(~13.86°@2006)。

> 核对:`Herakleios/main`(荀爽分支)与 `main` 此段**逐字节相同** → 是长期错配,不是近期被改坏。引入提交 `ed5b160 / e9a7f6a / 7c7e05f`(早期导入)。

## 三制正确口径(经参考盘逐颗实拍核对到角分)

参考盘 DHX:2006-10-04 09:58 上海(121E28'12, 31N13'48)。

- **回归今制(mode 2)** = 28 距星活体 tropical 黄经。
  - 距星 J2000 赤道坐标见 `perchart.MOIRA_DISTAR_J2000`(28 行,自有恒星案)。
  - `_moira_distar_lon()`:proper motion(出生年差)+ **严格 IAU 岁差 (ζ, z, θ)** + 盘历元真黄赤交角(`swisseph.ECL_NUT`)→ tropical 黄经。
  - 逐宿不均匀(各距星实际位置)。高赤纬星(柳=λDra Dec+69°、张=χUMa Dec+47°、亢=λCen Dec−63°)必须严格岁差,线性近似会偏。
- **回归开禧（开禧历, mode 3）** = `MOIRA_KAIXI_STELLAR_DEGREES`(16.0…)+ `_moira_ayanamsha(jd)`。
  - ayanamsha:`SE_SIDM_USER`,基准历元 1300-01-01,基准黄经差 4.0°。
  - 对应参考软件的「显示开禧宿度」独立项(开禧历宿度),**不是**参考软件菜单里的「古制」(见文末澄清)。
- **恒星制（郑式, mode 4）** = `MOIRA_CURRENT_STELLAR_DEGREES`(15.9…)**原值**。
  - 该制盘本身 sidereal(`isZhengSidereal` → `zodiacal=SIDEREAL`),planets 已 sidereal,故宿度用原值(不再加 ayanamsha)。等价于 tropical planet + (15.9 + ayanamsha)。

**置宿坐标**:三制(2/3/4)均**沿黄道**(planet.lon vs star.lon);荀爽/斗柄(0/1)沿**赤经**(RA)。由 `setPlanetSu28(byLon=)` + `fillPlanetSu28(byLon=)` 控制。

## 命度红线 + 中心立命

- **命度红线**:`GuoLaoMoiraWheel.renderDegreeTicks` 的 `life-degree-marker`:在**宿度度数带(r4~r7)**与**年龄环(r10~r11)**各画一条 `RED` 短径向线,标注立命度,对齐参考软件。
- **中心立命标注**(2026-06-02 对齐参考软件做法):`renderStaticTwelve` 中心原为**硬编码**「七政 / 立命 / 度木」(「度木」写死、不随盘变)。现按真实命度计算 = **命度所在宿+七曜 / 入宿度 / 立命**(如「女土 / 三度 / 立命」「心月 / 六度 / 立命」),用 `stellarIndexForDegree(buildFixedStars(chart), life)` 定位宿、`cnDegree()` 转中文度数。与命度红线共用同一 `life`,内部一致。

## 「古制」≠「开禧」澄清(易混淆)

参考软件菜单里 **「古制」和「开禧」是两个不同的东西**:
- 参考「**古制 + 岁差校正**」(回归校正古宿制)= **郑氏恒星基值(15.9)+ 岁差** = 参考「**恒星制**」= Horosa **`恒星制郑式(mode4)`**(已逐颗对齐到角分)。
- 参考「**显示开禧宿度**」= **开禧历宿度(16.0)** = Horosa **`回归开禧(mode3)`**(+岁差)。

源码依据:参考软件 `ChartData` 的 house_mode(今制/古制)分支里,古制走 `sidereal_stellar_names`(=郑氏恒星 15.9),`compass_stellar_names`(开禧 16.0)是另一个 `pref_show_compass`「显示开禧宿度」叠加项。所以要对照参考的「古制+校正」请看 Horosa「恒星制郑式」,而非「回归开禧」。
